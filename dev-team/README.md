# dev-team — команда агентов полного цикла разработки

Проект-«команда» для работы над любым кодом по полному циклу SDLC. Главный агент —
**оркестратор**: он раскладывает задачу на этапы и направляет их специализированным
worker-субагентам, держит цикл правок и общается с пользователем. Сам он код не пишет,
не тестирует и не ревьюит — это делают воркеры.

Поддержка языков: Python, TypeScript/Node, Go (через скиллы стандартов под каждый язык,
которые подгружаются по типам файлов). Общение с пользователем — на русском; файлы агентов,
скиллы и код — на английском.

## Цикл работы
```
explore → architect → implement → test/run → (debug если красное)
        → code-review ∥ security-review → document
```
Параллелизм: разведка файлов веером; `test-author` параллельно с `implementer` после фиксации
интерфейсов; `code-reviewer` и `security-reviewer` одновременно на одном диффе. Оркестратор
владеет циклом «правка → перепрогон → переревью» и эскалирует пользователю только настоящую
неоднозначность.

## Команда (9 воркеров)
| Агент | Модель | Вход → выход |
|---|---|---|
| `architect` | opus | задача + карта кода → план (изменения, интерфейсы, риски) |
| `codebase-explorer` | haiku | вопрос + путь (только чтение) → карта файлов и поведения |
| `implementer` | sonnet | план + файлы → дифф |
| `test-author` | sonnet | план/интерфейсы → тесты |
| `test-runner` | haiku | дифф + тесты → отчёт pass/fail (без правок кода) |
| `debugger` | opus | падающие тесты + дифф → корневая причина + минимальный фикс |
| `code-reviewer` | sonnet | дифф → замечания по качеству/читаемости/простоте |
| `security-reviewer` | opus | дифф + зависимости → находки по безопасности (OWASP) |
| `documenter` | sonnet | финальное изменение → обновлённые доки/changelog |

opus стоит только там, где это оправдано (проектирование, root-cause, состязательный
security-review); остальные — sonnet/haiku по нагрузке (см. `proj_arch/patterns/model-tiering.md`).

## Скиллы
- **Bundled (переиспользуются):** `/code-review`, `/debug`, `/run`, `/verify`.
- **Авторские (`.claude/skills/`):** `security-review`, `simplify`, `test`.
- **Под язык (`.claude/skills/`):** `python-standards`, `typescript-standards`, `go-standards` —
  подгружаются по описанию только для своего языка.

## Структура
- `CLAUDE.md` — оркестратор: роль, модели, цикл, таблица маршрутизации, память.
- `.claude/agents/` — девять файлов воркеров (английский, без инструмента `Agent`).
- `.claude/skills/` — авторские и языковые скиллы.
- `.claude/settings.local.json` — обычные безопасные права; запись вне корня проекта
  (в целевой репозиторий) спрашивается подтверждением.
- `.mcp.json` — `sequential-thinking`, `fetch`; общие серверы доступны через
  `enableAllProjectMcpServers`.
- `work/` — площадка обмена между воркерами (`plan.md`, сводки диффов, отчёты прогонов).

## Память
Тег проекта: `dev-team`. На старте сессии оркестратор запрашивает память
(`recall_memories` + `search_memories(tags=["dev-team"])`). Важно: в этом инстансе memorygraph
тип `decision` не поддерживается — решения сохраняются как `type=code_pattern` с тегом
`decision` (см. секцию `## Memory` в `CLAUDE.md`).
