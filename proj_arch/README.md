# proj_arch — архитектурный мозг и фабрика проектов

`proj_arch` — центральный проект, который управляет всей экосистемой агентов Claude в
`~/Documents/claude/`. Делает три вещи:

1. **Хранит канон** — паттерны агентной разработки в `patterns/`, по которым строится любой
   новый проект.
2. **Генерирует проекты и агентов** — по запросу создаёт новый проект целиком (папка,
   `CLAUDE.md`-оркестратор, субагенты, `.mcp.json`, права) либо отдельного субагента.
3. **Ведёт реестр** — `REGISTRY.md` + зеркало в memorygraph: какие проекты и агенты есть,
   на каких моделях, с какими MCP.

## Три сквозных правила (канон для всего флота)

1. **Главный агент любого проекта — только оркестратор.** Вся работа — в субагентах.
2. **Skills и файлы агентов пишутся на английском.** Общение с пользователем и публикуемый
   контент — на целевом языке проекта (здесь русский).
3. **Память — проактивная.** Каждый агент запрашивает memorygraph до работы и сохраняет
   выводы после.

## Как пользоваться

Запусти сессию Claude Code в этой папке и сформулируй запрос на русском — оркестратор сам
выберет нужного воркера:

- **«создай проект под задачу X»** → `project-scaffolder` строит весь проект из канона.
- **«добавь субагента Y в проект Z»** / **«напиши субагента…»** → `agent-author` пишет один
  файл агента по канону.
- **«проверь/обнови паттерны»**, **«добавь паттерн про…»** → `pattern-curator` сверяет канон
  с актуальным Claude Code и правит `patterns/`.
- **«обнови реестр»** → `registry-keeper` приводит `REGISTRY.md` и memorygraph в соответствие
  с диском.

## Структура

```
proj_arch/
├── CLAUDE.md            # оркестратор (английский; общение с пользователем — русский)
├── README.md           # этот файл
├── REGISTRY.md         # реестр проектов и агентов флота
├── .mcp.json           # MCP-серверы
├── .claude/
│   ├── settings.local.json   # права: RW + защиты (см. patterns/permissions-safety.md)
│   └── agents/               # 4 воркера-фабрики
└── patterns/           # канон (по одному правилу на файл + README-индекс)
    templates/          # рыбы, по которым генерируются новые проекты
```

## Доступ

`proj_arch` — привилегированный проект: читает `~/.claude/` и все проекты, может создавать и
править чужие проекты и настройки Claude. Опасные операции защищены: запись в чужой проект или
в `~/.claude/` требует подтверждения (`ask`), деструктивные команды (`rm -rf`, force-push и
т.п.) запрещены (`deny`). Подробности — в `patterns/permissions-safety.md` и
`.claude/settings.local.json`.

## Память

Project tag: `proj_arch`.
На старте сессии оркестратор запрашивает memorygraph по тегу `proj_arch`
(`recall_memories` + `search_memories`) — прошлые решения, состояние реестра, антипаттерны.
Записи, касающиеся конкретного управляемого проекта, кросс-тегируются и его тегом
(например `["proj_arch","rnd"]`).
